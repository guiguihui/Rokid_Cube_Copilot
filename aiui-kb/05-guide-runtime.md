# 05 · 运行时

源：`docs/aiui/0-guide/runtime/*`

---

## env — 运行环境
| 环境 | 特点 |
|------|------|
| **Rokid Glasses(乐奇眼镜)** | 逻辑层+视图层跑在自研容器 **Ink**（基于 **QuickJS + Skia**），针对穿戴设备极低功耗/内存/CPU 深度优化 |
| **Craft** | 基于 Web 的 AIUI 开发 IDE：代码编辑/工程管理/智能体绑定/实时预览 |
| **Rokid AR Lite / AR Studio** | 计划中，跨 AI+AR 设备一致体验 |

平台差异：
1. **硬件/传感器**：DevTools 模拟器中 6DoF 追踪、手势识别、空间音频、传感器多为模拟数据 → **必须真机验证**。
2. **性能**：桌面端资源充裕；穿戴设备严格限功耗发热，复杂动画/高频 `setData` 在真机可能触发瓶颈或降频。
3. **渲染排版**：不同分辨率/FOV 物理设备上空间比例、深度、排版有细微差异。
- ⚠️ DevTools 仅供开发调试，最终表现以 **Rokid 真机为准**。

---

## js — JavaScript 兼容性
- Ink 的 JS 引擎 = **QuickJS（版本 2024-01-13）**，轻量、完整支持 **ES2023**。
- 支持：`Promise`/`async-await`（无需 polyfill）；`Map`/`Set`/`WeakMap`/`WeakSet`；可选链 `?.` 与空值合并 `??`；ES Modules(`import`/`export`)；`BigInt`(如 `9007199254740991n`，适合高精度时间戳/硬件标识)。

---

## mechanism — 运行机制（5 步）
1. 启动 → 初始化逻辑层
2. 逻辑层加载首页数据 → 发给视图层
3. 视图层初次渲染
4. 用户交互 → 视图层把事件传给逻辑层
5. 逻辑层处理事件、更新数据 → 驱动视图层重渲染

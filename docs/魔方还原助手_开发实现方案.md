## 0. 目标与约束（已与用户敲定）

- **目标**：3 小时内开发完一个 standalone AIUI agent，闭环 `输入魔方状态 → 求解 → HUD 逐步箭头引导 → 完成`，随后在 PC 预览测试，再上真机。

## 3. 核心数据模型与约定

### 3.1 标准配色（西方配色，固定，中心格锁定）
六个面的中心颜色固定，建立 **颜色 ↔ 面字母** 双射：

| 面字母 | 含义 | 中心颜色 | 颜色字母 |
|--------|------|----------|----------|
| U | 上/顶 | 白 | W |
| D | 下/底 | 黄 | Y |
| F | 前 | 绿 | G |
| B | 后 | 蓝 | B |
| L | 左 | 橙 | O |
| R | 右 | 红 | R |

```js
// lib/cube.js
export const COLORS = ['W','Y','R','O','G','B'];      // 网格里存这些
export const COLOR_TO_FACE = { W:'U', Y:'D', R:'R', O:'L', G:'F', B:'B' };
export const FACE_TO_COLOR = { U:'W', D:'Y', R:'R', L:'O', F:'G', B:'B' };
export const FACE_NAME_CN  = { U:'顶面', D:'底面', F:'前面', B:'后面', L:'左面', R:'右面' };
```

### 3.2 网格结构（输入页用）
```js
// grid: 每面 9 格（index 0..8 = 从左上到右下行优先），index 4 = 中心(锁定)
grid = { U:[...9], R:[...9], F:[...9], D:[...9], L:[...9], B:[...9] }  // 元素为颜色字母
// 初始(已还原态)：每面 9 格全为该面中心色
```

### 3.3 facelet 串（cubejs 输入，54 字符）
cubejs 采用标准 Kociemba 布局：顺序 **U R F D L B**，每面 9 格行优先，字符是**面字母**。
```js
// 颜色网格 → facelet 串（把每格颜色字母换成面字母，按 URFDLB 拼接）
export function gridToFacelets(grid){
  return ['U','R','F','D','L','B']
    .map(f => grid[f].map(c => COLOR_TO_FACE[c]).join(''))
    .join('');
}
// facelet 串 → 颜色网格（反向，给「载入测试打乱」用）
export function faceletsToGrid(s){
  const order=['U','R','F','D','L','B'], grid={};
  order.forEach((f,k)=>{ grid[f]=s.slice(k*9,k*9+9).split('').map(ch=>FACE_TO_COLOR[ch]); });
  return grid;
}
```

### 3.4 校验
```js
export function validate(grid){
  const errs=[];
  // 1) 中心锁定且符合配色
  for(const f of ['U','R','F','D','L','B'])
    if(grid[f][4]!==FACE_TO_COLOR[f]) errs.push(`${FACE_NAME_CN[f]}中心色应为${FACE_TO_COLOR[f]}`);
  // 2) 每种颜色恰好 9 个
  const cnt={}; COLORS.forEach(c=>cnt[c]=0);
  for(const f of ['U','R','F','D','L','B']) grid[f].forEach(c=>{ if(c) cnt[c]++; });
  COLORS.forEach(c=>{ if(cnt[c]!==9) errs.push(`颜色 ${c} 有 ${cnt[c]} 个(应为9)`); });
  // 3) 是否有空格
  for(const f of ['U','R','F','D','L','B']) if(grid[f].some(c=>!c)) errs.push(`${FACE_NAME_CN[f]}有未填格`);
  return errs; // 空数组=合法
}
```
> 完整可解性（角/棱朝向奇偶）交给 cubejs：`solve()` 对非法态会抛错，捕获后提示「状态不合法，请检查」。

## 4. 求解器封装（`lib/solver.js`，最高风险，最先做）

### 4.1 cubejs 接口（确认用法）
- `Cube.initSolver()` —— 构建剪枝表，约 4–5s，**幂等**。
- `new Cube()` —— 已还原态；`cube.move("U R2 F'")` 应用走步；`cube.isSolved()`；`cube.asString()` → facelet 串。
- `Cube.fromString(facelets)` —— 从 54 字符串建魔方；`.solve()` → `"U R2 F' ..."`（约 ≤22 步）。

### 4.2 封装
```js
import Cube from './vendor/cube.solve.js';
let inited=false;
export function initSolver(){ if(!inited){ Cube.initSolver(); inited=true; } }
export function solve(facelets){ initSolver(); return Cube.fromString(facelets).solve(); }

// 测试打乱：存走步串，运行时生成（避免手写 54 字符）
export const SCRAMBLE = "R U R' U R U2 R' F R F'";   // 可换更乱的
export function scrambledFacelets(){
  initSolver(); const c=new Cube(); c.move(SCRAMBLE); return c.asString();
}
// dev 自检：求解 → 回放 → 必须还原
export function selfTest(){
  initSolver();
  const c=new Cube(); c.move(SCRAMBLE);
  const sol=c.solve(); c.move(sol);
  const ok=c.isSolved();
  console.log('[selfTest]', ok?'PASS':'FAIL', 'sol=',sol);
  return ok;
}
```

### 4.3 适配 Ink/rquickjs 沙箱
- cubejs 若是 UMD/浏览器全局，需改成 ES 导出：文件末尾确保 `export default Cube;`（必要时去掉对 `window`/`self` 的依赖，参考 `samples/scanner/lib/webp.js` 里的沙箱包裹手法）。
- **时间盒 40 分钟**：先让 `selfTest()` 在 `aiui run --debug` 控制台打出 PASS，再往下做。若卡死：退化方案见 §9。

## 5. 走步解析（`lib/moves.js`）

把 `"R U' F2 ..."` 解析为新手可懂的步骤数组，喂给引导页与 canvas。

```js
import { FACE_NAME_CN } from './cube.js';
// 单步：{face, name, dir:'CW'|'CCW'|'180', dirText, deg}
export function parseMoves(str){
  return str.trim().split(/\s+/).filter(Boolean).map(tok=>{
    const face=tok[0], mod=tok.slice(1);
    const dir = mod==="'" ? 'CCW' : mod==='2' ? '180' : 'CW';
    const dirText = dir==='CW'?'顺时针':dir==='CCW'?'逆时针':'转180°';
    return { face, name:FACE_NAME_CN[face], dir, dirText, deg: dir==='180'?180:90 };
  });
}
export function stepText(s){ return `${s.name} ${s.dirText}`; }  // 例:「右面 顺时针」
```
> 方向约定：**正对着该面看**，CW=顺时针。引导页/语音都用此约定，务必一致。

## 6. 输入页 `pages/input/index.ink`

### 6.1 职责
- 渲染 6 面颜色网格（中心锁定），可改色、可一键载入测试打乱、可求解。
- 求解成功 → 走步串存 `localStorage` → 跳引导页。
- 预留「拍照填充」按钮（真机加分项）。

### 6.2 data 与方法
```js
data: {
  grid: SOLVED_GRID(),          // 见 §3.2 初始态
  focus: { face:'U', i:0 },     // 当前选中格
  status: 'idle',               // idle|solving|error
  errorText: '',
  flat: [],                     // 渲染用：见 6.3
}
methods:
- buildFlat(): 把 grid 摊平成 [{face,i,color,locked,sel}] ×54，setData 到 flat（规避模板复杂表达式）
- onCellTap(e): 读 dataset.face/.i → 若 locked 忽略；否则设 focus 并循环切色，重建 flat
- cycleColor(face,i): grid[face][i] = 下一个颜色(W→Y→R→O→G→B→W)
- onKeyDown(e): Enter=求解；方向键移动 focus；空格=切色；Backspace=exit
- loadScramble(): faceletsToGrid(scrambledFacelets()) → setData → buildFlat
- doSolve(): const errs=validate(grid); 有错→status='error',errorText; 无错→try{ const mv=solve(gridToFacelets(grid)); localStorage.setItem('cube.moves', JSON.stringify(mv)); wx.navigateTo({url:'pages/guide/index'}) }catch{ 提示不合法 }
- takePhotoFill(): (真机) takePhoto→decodeWebP→colors.classifyFace→填一面→buildFlat
```

### 6.3 渲染（规避嵌套 ink:for）
两种做法二选一：
- **A（推荐）**：用单个 `flat`(54 元素) + CSS grid 一次性铺；每格 `data-face`/`data-i`，class 据 `locked/sel/color` 变化。
- **B**：6 个平级 `ink:for`，每个渲染该面的 9 格。
```html
<page>
  <view class="root">
    <view class="bar">
      <text class="title">输入魔方状态</text>
      <view class="btn" bindtap="loadScramble">载入测试打乱</view>
      <view class="btn primary" bindtap="doSolve">求解</view>
    </view>
    <text class="err" ink:if="{{status==='error'}}">{{errorText}}</text>

    <!-- 做法A：单网格，54 格 -->
    <view class="cube">
      <view ink:for="{{flat}}" ink:key="*this"
            class="cell color-{{item.color}} {{item.locked?'lock':''}} {{item.sel?'sel':''}}"
            data-face="{{item.face}}" data-i="{{item.i}}" bindtap="onCellTap">
        <text>{{item.color}}</text>
      </view>
    </view>
  </view>
</page>
```
> 单绿屏不能显真彩，**格内用颜色字母 W/Y/R/O/G/B**；选中格 `sel` 用亮边框；锁定格 `lock` 半亮。`color-*` 类可仅作不同亮度/边框区分（非真彩）。布局用 CSS grid 把 54 格摆成展开十字或 2×3 面阵，MVP 摆成 6 个 3×3 块即可（参 §3 顺序）。

### 6.4 样式要点
- 黑底绿前景；`.cell` 36–44px 方格、绿边框 2px、字号 18px、居中；`.sel` 边框 `#40ff5e` 加粗、`.lock` 文字 `rgba(64,255,94,.6)`。
- `.btn` 绿边框胶囊；`.primary` 反白底绿。

## 7. 引导页 `pages/guide/index.ink`

### 7.1 职责
读走步 → 逐步用 canvas 画「面 + 旋转箭头 + 步数」，语音播报，键盘/语音上下步，完成统计。

### 7.2 data 与生命周期
```js
data:{ steps:[], idx:0, total:0, done:false }
onLoad(){
  const mv = JSON.parse(localStorage.getItem('cube.moves')||'[]');
  const steps = parseMoves(mv.join ? mv.join(' ') : mv); // mv 存的是数组或串
  this.setData({ steps, total: steps.length, idx:0, done:false });
}
onReady(){ this.renderStep(); this.speakStep(); }   // canvas 须在视图就绪后画
methods:
- renderStep(): 取 ctx=selectComponent('#hud').getContext('2d') → 清屏 → 画顶部 idx+1/total → 画中央面方块+字母 → 画旋转箭头(见 7.3)
- speakStep(): speechSynthesis.speak(new SpeechSynthesisUtterance(stepText(steps[idx])),'immediate')
- next(): idx<total-1? idx++ →render+speak : finish()
- prev(): idx>0? idx-- →render+speak
- repeat(): render+speak
- finish(): setData{done:true}; 画完成页+用时/步数; new Sound('/assets/done.wav').play()
- onKeyDown(e): Enter=next; Backspace=prev(或 done 时 exit)
```

### 7.3 Canvas 箭头绘制规格
画布 `<canvas id="hud" width="480" height="300">`，坐标如下（绿色 `#40ff5e`）：
```
顶部: fillText(`${idx+1} / ${total}`, 居中, y=36)，字号 24
中央: 在 (cx=240, cy=160) 画 120×120 圆角方块(strokeRect/或路径)，
      内部 fillText(面字母, 居中, 大字号 56)
箭头: 围绕方块画一段弧 + 箭头头部：
   - CW : ctx.arc(cx,cy,90, -0.3π, 1.1π, false) 末端画箭头(两条短线)
   - CCW: ctx.arc(cx,cy,90,  1.3π,-0.1π, true ) 末端反向箭头
   - 180: 画整圈虚线 + 中央叠加「×2」或上下双箭头
底部: fillText(stepText(step), 居中, y=288)  // 例「右面 顺时针」
```
箭头头部：在弧线终点按切线方向画两条回折短线（长 14px，夹角 ±150°）。`ctx.lineWidth=6; ctx.lineCap='round'`。

### 7.4 完成态
显示「完成！共 {{total}} 步」+（可选用时）；按键退出。

## 8. 语音命令（`pages/guide/index.ink`，2:40 后接）

```js
startVoice(){
  this.rec = new SpeechRecognition();
  this.rec.onresult = e => { this.handleVoice(e.results[0][0].transcript||''); };
  this.rec.onend = () => { if(!this.data.done) this.rec.start(); };  // 持续监听
  this.rec.onerror = () => {};
  this.rec.start();
}
handleVoice(t){
  if(/下一?步|下个|next|继续/.test(t)) this.next();
  else if(/上一?步|返回|back/.test(t)) this.prev();
  else if(/重复|再说|repeat/.test(t)) this.repeat();
}
```
> 键盘/点击始终是兜底，语音失败不影响主路径。`onHide/onUnload` 里 `this.rec && this.rec.abort()`。

# 真机自动化测试流程（adb 自驱，无需 Craft / 无需戴眼镜）

> 用途：纯 PC 用 adb **部署 + 驱动 + 观察** 眼镜上的魔方 agent，做端到端回归。已在 2026-06-18 实测跑通。
> 配套：部署/调试链路见记忆 `rokid-glasses-realdevice-debug`、`rokid-glasses-deploy-adb`。

## 能做 / 不能做（一句话）
- ✅ 部署代码（adb push）、驱动 UI（keyevent）、观察结果（截屏/日志/存储）、跑完整「求解→引导」happy path。
- ⚠️ **改了代码后要"全新启动"才生效**，而全新启动目前只能语音「打开魔方还原助手」（resume 不重载，见 §2）。
- ❌ 真实相机无法自动喂魔方 → 用「测试图片」(base64 离线) 替代（见 §5-B）。

## 1. 部署（push 代码）
```bash
RT="/sdcard/jsai/runtime/617baca8416b4e49a37c51394b89b418_1.0.15_d471cf9b"
"$ADB" push E:/ROKIT/lib/solver.js        "$RT/lib/solver.js"
"$ADB" push E:/ROKIT/pages/input/index.ink "$RT/pages/input/index.ink"
# 只推改动的文件即可。验证：
"$ADB" shell "grep -n 'const VERSION' $RT/pages/input/index.ink"
```
- 运行时目录**可写**（实测 WRITE_OK）；设备非 root，但 /sdcard 是外部存储 shell 可写。

## 2. 让新代码生效（⚠️ 唯一需要语音的一步）
- **现状**：新 JS 只在 **全新启动(`app.onLaunch`)** 时加载。从助手 feed 回到 agent 是 **resume(`onShow`)**，**不重载 JS**（logcat 见 `[cube-coach] show` 而非 `launch`）。
- **全新启动** = 语音「**打开魔方还原助手**」/「开心开心」。这步当前要人/语音（未找到 adb 注入 ASR 的办法）。
- **确认已加载新版**：截屏看首页右上角版本号（`v1.0.x`）。
- 若本次只测**已加载版本的行为**（不改代码）→ 跳过本步，直接 §3。

## 3. 驱动（键盘事件 → app onKeyDown）
```bash
"$ADB" shell input keyevent KEYCODE_DPAD_RIGHT   # 实测 → app 收到 'ArrowRight'
```
| keyevent | app event.code | 作用 |
|----------|----------------|------|
| `KEYCODE_DPAD_DOWN`/`_RIGHT` | ArrowDown/ArrowRight | 向前（焦点+1） |
| `KEYCODE_DPAD_UP`/`_LEFT` | ArrowUp/ArrowLeft | 向后（焦点-1） |
| `KEYCODE_ENTER` | Enter | 单击（确认/切色） |
| `KEYCODE_BACK` | Backspace | 返回（首页=退出到助手） |
- ⚠️ **滑动有 250ms 节流**：连发方向键要 `sleep 0.3` 以上，否则被当成一次。

## 5. 现成端到端脚本

### A. 求解→引导（happy path）✅ 全自动，无需相机/网络/重载
前提：agent 已在首页（焦点 0=「扫描魔方」）。
```bash
"$ADB" shell input keyevent KEYCODE_DPAD_RIGHT; sleep 0.4   # 焦点→载入测试打乱
"$ADB" shell input keyevent KEYCODE_ENTER;      sleep 0.6   # 载入打乱(焦点自动→求解)
"$ADB" shell input keyevent KEYCODE_ENTER;      sleep 3.5   # 求解→引导页
# 验证：storage.json 的 cube.moves 非空；截屏见 "1/N · 某面 顺/逆时针" + 箭头
```
- 引导步进：`KEYCODE_ENTER` = 下一步。
- 「载入测试打乱」循环 6 个态（4/5/20/21/21/21 步），多点几次换不同情形。
- 实测：得 `cube.moves="R L' D R2"`，引导页正确显示「1/4 · 右面 顺时针」+ 旋转箭头。

### B. 扫描→复核（测试图片，免相机）需先重载（§2）
「测试图片」已改 **base64 内嵌离线**（`lib/testData.js` 由 `tools/bake-testdata.mjs` 从 `testImg/*.png` 生成；真机 `fetch` 不通）。重载新版后：
```bash
# 在扫描页(aim) focus 0=拍照：
"$ADB" shell input keyevent KEYCODE_DPAD_RIGHT; sleep 0.4   # 焦点→测试图片
"$ADB" shell input keyevent KEYCODE_ENTER;      sleep 2.5   # 加载+识别→复核页(缩略图+9格)
# 截屏应见"刚拍到的画面"缩略图 + 中文色名 9 格；接受本面→下一面，6 面后回首页
```

## 5.1 可靠性：多步要逐步验证（实测教训）
- **别盲打长序列**：连发十几个 keyevent 不看中间态，一旦某步时序/焦点没对上就全乱（实测 6 面盲序列跑偏）。**每个关键步后 `screencap` 读图确认**再发下一步。
- **外部事件会打断**：手机 App 蓝牙断开会把眼镜带回 launcher 主界面（提示"手机 App 已断开"），中断自动序列。跑长流程前确认手机连着。
- **单面已稳定验证**：扫描页→测试图片→复核(缩略图+9格识别)，单步驱动 100% 可复现。

## 6. 已知限制
- **重载需语音**（resume≠reload）：改代码后这一步要人。待研究：ASR 注入 / 直接 launch agent 的 intent。
- **真实相机**：`takePhoto` 抓镜头前实景，无法自动喂魔方 → 用「测试图片」(base64) 替代。
- **截屏**：是显示画面；`FLAG_SECURE` 不阻断 `screencap`/scrcpy（实测）。
- **scrcpy 投屏**（给人看/手控）：`$env:ADB=<adb全路径>; scrcpy -s 1901092524001700 -w --window-title=Rokid-Glasses`（**别加 `--no-audio`**，该 v4.0 不认会退出）。

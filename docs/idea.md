# 产品创意池（AIUI / Rokid Glasses）

> 目标：在 Rokid Glasses + AIUI 这块硬件上，做**一个**令人印象深刻、痛点真实普遍、且尚未被人占据的小功能点。
> 硬约束（每个点子都要过这一关）：常驻免手 / 语音进 + 语音出 / 云端流式大模型 / 单色绿小字 HUD（几个字+箭头，无彩色无富 AR）/ 按需拍**单张**照片（无连续视频）/ 麦克风 + BLE + IMU。
> 做不了：连续实时视觉、彩色显示、富 AR 配准、重本地计算。

---

## 候选 1：「接话器」——抓住别人刚说的关键信息

**一句话**：有人快速报了手机号 / 地址 / 时间 / 名字，话音一落就没了；按一下 Ring（或一句口令），眼镜把刚才那句里的**关键信息**抽出来，几个字钉在视野角落，念回去确认。

**四标准评估**
- **真实+普遍**：美国人平均一年问 1095 次"你说啥"（每天 3 次），且只敢问两次就放弃假装听懂。记忆铁律：断了复述后信息留存 3 秒 80% → 18 秒 <10%。
- **可演示**：对方报号码 → 瞥一眼 → 号码在眼前，免手免纸笔，"哇"点直接。
- **硬件契合**：只吃长板（麦克风 + STT + 大模型抽取 + 一瞥单色字）。单色小 FOV 只显几个字，对这个功能恰是优点。
- **白地**：主流眼镜全是正向滚动实时字幕；录音类（Limitless/Plaud/Bee）全是事后在手机看。"当场、免手、抽取式"是空的。

**MVP 形态**：按需触发（按 Ring / 说"记一下"），只回看刚才十几秒，不常驻偷听 → 绕开两方同意法（11–13 个州，连不存盘转写都可能算录音）+ 省电。把"只记 30 秒就忘"做成卖点。

**⚠️ 已识别的硬伤（为什么暂缓 / 需要再找）**
1. **没充分发挥眼镜形态优势**：这个功能本质是"录音 + 抽取"，一个耳机/胸针/手机都能做，眼镜形态（第一人称视觉、HUD）没用上，护城河弱。
2. **收音方向错位**：眼镜的麦克风戴在**自己**身上，对**自己**的声音收得最好，对**对面别人**的话收音不一定稳——嘈杂环境、距离、朝向都会导致信息缺失，而"漏一位数字"就废了。

> 结论：痛点成立，但与产品形态的契合度存疑。继续寻找**真正吃眼镜独有优势**的痛点（第一人称"看我所看" + 解放双手 + 一瞥即得 + 麦克收自己的音最准）。

## 形态优势这把尺子

眼镜相对手机/耳机的真正独有优势：
1. **第一人称摄像头**——看到的正是你正在看 / 正在做的东西（按需单张照片即可）。
2. **戴在脸上的 HUD**——信息直接进视线，不用低头看手机。
3. **彻底解放双手**——双手在干活时仍能获取信息。
4. **麦克对"自己"的声音收得最准**——适合"我说"，不适合"听别人说"。

> 一个点子越是"非眼镜不可"（尤其必须用第一人称摄像头），护城河越深。下面用这把尺子给候选排序。

| 候选 | 靠摄像头(只眼镜能做) | 靠收别人音(弱点) | 耳机也能做? | 白地 |
|---|---|---|---|---|
| **A 「适合我吗」标签速查** | ✅ 必须第一人称拍 | ❌ 不需要 | ❌ 做不了 | ✅ 确认空 |
| B 菜谱步骤定位器 | ⚠️ 仅录入拍一次 | ❌ 不需要 | 部分 | ⚠️ 半空 |
| C 「我做了吗」面包屑 | ❌ 不用 | ❌ 不需要 | ✅ 大半能 | ✅ 确认空 |
| D 名字/词怎么念 | ❌ 不用 | ❌ 不需要 | ✅ 大半能 | ✅ 确认空 |

## ⭐ 候选 A（推荐）：「这个适合我吗」——个人化标签速查

**一句话**：扫一眼任何食品/商品标签 → 拍一张 → 眼镜对照**你自己**设的忌口清单（过敏原 / 麸质 / 猪肉明胶酒精 / 你讨厌的成分 / 高糖高钠），语音念出"含花生 ✗"，HUD 上钉那个犯规的词。

**四标准**
- **真实+普遍**：美国 10.8% 成人 + ~8% 儿童食物过敏；过敏成人 51% 经历过严重反应，食物过敏急诊 10 年涨 377%。叠加素食/清真/犹太洁食/无麸质/控糖控钠，几乎人人沾边。需求已被验证到海量级：**Yuka 8000 万用户、83 亿次标签扫描**（仅 2024 就 27 亿）——人们本来每天就在对着标签举相机。
- **可演示**：超市里拿起一包零食 → 瞥一眼 → "含花生 ✗"。"哇"点直接，且免手（手正握着商品）。
- **硬件契合 + 形态独占**：单张照片 → 云端 LLM 对照忌口清单 → 输出最小（一句话 + 一个 ✓/✗ 字形）。**必须用第一人称摄像头，耳机/胸针做不了**，正好补上"接话器没发挥形态优势"的洞；且完全不碰"收别人音"的弱点；忌口清单用自己语音设一次即可。
- **白地**：确认无任何在售眼镜做**个人化**"适合我吗"判断——大家只会**通用地把标签念出来**（Ray-Ban Meta / Envision / Rokid 的 OCR）。这是对 Yuka 的严格升级（免手、进视线、不掏手机）。可自然延伸到清真/洁食/素食。
- **护城河注意**：通用 OCR 念标签不是新东西，本功能的壁垒在**个人化 + 可靠 + 一瞥**，不在"OCR 本身新"。

## 候选 B：菜谱步骤定位器——"我做到第几步了"

**一句话**：手沾着生鸡肉/面糊不能碰手机，屏幕又睡了；说"下一步 / 第几步"，下一行菜谱就出现在眼前。开始前拍一张菜谱，LLM 拆成步骤，全程语音推进。

- **真实+普遍**：SafeHome 调研（n=1986）96% 做饭时分心、59% 边做边看手机、超 2/3 不固定用计时器；编辑实测痛点直白——"手上全是生鸡肉，得先洗手才能唤醒屏幕"。做饭是周/日级高频习惯。
- **硬件契合**：一行字 + 步骤 N/M 正是单色 HUD 最擅长；"下一步/上一步/重复/几度"是自己对自己说；拍一张完成录入。
- **白地（半）**：Meta/Solos 有"对话式"做饭帮助，**Meta Ray-Ban Display 已能手动翻步骤卡**——所以"眼镜上显步骤"不再全空。**但没人做"确定性记住你在第几步"**；Meta Connect 2025 现场翻车正栽在这（AI 说"你已混合好基底"，主厨面前却是空碗）。壁垒=**确定性记位 + 拍你自己的菜谱**，不是再来一个话痨 AI。

## 候选 C：「我做了吗」——日常动作语音面包屑

**一句话**：出门/睡前想不起锁没锁门、吃没吃药、关没关火；顺嘴说一句"锁了门""吃了药"，过会儿问"我吃药了吗"→ HUD 显"门：20:42 已锁"。

- **真实+普遍**：忘记上一次服药是误重复用药的常见原因；~50% 慢病患者不按时服药，忘记是首因。"锁没锁/关没关火"的反复怀疑是教科书级强迫；实验证明**反复检查本身会侵蚀记忆信心**——怀疑才是痛点，一条带时间戳的外部记录就能终结它。
- **硬件契合**：纯自己语音捕捉，无需拍照、无需听别人；召回是一行带时间戳的字，单色 HUD 完美。
- **白地**：确认空——Meta 做的是"将来"提醒（"提醒我给妈发消息"），没人做"过去完成确认"。
- **弱点**：① 不用摄像头，**耳机也能做大半**，形态护城河弱；② 要靠你**记得说**那一句（吃药场景最自然，因为动作本身会提醒）。

## 候选 D：「这名字/词怎么念」——开口前的私密发音提示

**一句话**：要叫一个不确定怎么念的名字（Saoirse / Nguyen）或点一道菜（Gewürztraminer）前，悄声问一句，眼里弹出 1–2 个字的音标"SEER-sha"，可选 TTS 在自己耳边先念一遍。

- **真实+普遍**：NameCoach 调研 74% 在职场被名字发音难倒；**22% 因怕念错而回避介绍他人、16% 回避跟同事说话、13% 回避在会上点名**。OpenTable 调研 1/5 的人不敢点看不懂的菜。
- **硬件契合**：自触发的固定时刻 + 自己语音问 + 私密一瞥（别人看不到）→ 正中"悄声进、私密出"。无需摄像头、无色、几个字。框架健康（包容/尊重），躲开"AI 作弊"的舆论坑。
- **白地**：确认全部在售眼镜都没做。
- **弱点**：同样不用摄像头，**耳机也能做大半**；痛点偏窄（虽情绪强）。

## 推荐

主推 **候选 A「这个适合我吗」标签速查**。理由：四个里**只有它真正"非眼镜不可"**（强制第一人称摄像头），直接补上你指出的形态优势缺口；完全不依赖收别人的音；单张照片 + 一句话 + 自己语音设一次清单，全在硬件能力内；确认白地且 demo 一秒见效；还能从过敏延伸到清真/洁食/素食/控糖等海量人群。

次选 **候选 B 菜谱步骤定位器**（最高频、最普适，但被 Meta Display 翻卡功能部分蚕食，壁垒押在"确定性记位"）。C、D 是干净白地、情绪强，但都不用摄像头、耳机能做大半，**形态护城河偏弱**——正是上一轮接话器栽过的同一个坑，故降级。

# 第二轮：用"手机真的做不了"这把尺子重搜（候选 A 已废）

> 候选 A「标签速查」被否：扫标签时你站着不动、一只手空、能随时暂停——正是手机主场，眼镜只省"掏手机低头"一下，撑不起产品。和接话器栽在同一处。
> 新尺子：**只做"掏手机这个动作本身被禁止"的场景**——双手+眼睛都被现实任务占住、不能暂停。再加一道更狠的：**连耳机都替代不了（非得有摄像头 / 眼前 HUD）**。
> 开了 5 个 agent 各钉一个"手机做不了"的窄带，各自带回最优解 + 证据 + 白地核查：

| 区 | 最优解 | 手机真做不了? | 耳机也做不了?(非得看/摄像头) | 市场 | 白地 |
|---|---|---|---|---|---|
| **演奏/登台** | **歌词/和弦提词**(Ring/脚踏翻页) | ✅ 手在琴上眼在观众 | ✅✅ **必须看**(在唱歌，音频会撞) | 中大(光敬拜乐队 23 万教会) | ✅ 提词器全瞄演讲者 |
| **当众讲话** | **隐形计时器**("还剩2分") | ✅ 看表=破坏眼神接触 | ⚠️ 音频也能提醒 | **极广**(TED/会议/敬酒) | ✅ 各家提词器都没计时 |
| **手+工具干活** | **拍电阻→报数值** / **报尺寸悬浮到下刀** | ✅ 手脏/满、眼在工件 | 输入侧✅(耳机没相机) | 中(电子/木工/维修) | ✅ 消费级无人做 |
| **双手满的日常** | **3am 喂奶提示**(左/右+距上次) | ✅✅ 抱娃+黑灯+缺觉(最强) | ✅ 黑灯要看、不能出声吵娃 | 窄但极深(哺乳父母) | ✅ 没眼镜跟踪喂奶侧 |
| **开车/骑行** | 限速提示 | ✅ 但**法律/分心高危** | — | 广 | ⚠️ 已饱和、数据差；困倦检测此硬件**测不了** |

**关键洞察**：开车区排除——法律分心高危、数据质量差，且困倦检测这硬件物理上做不了（摄像头朝外、无连续视频、算不出 PERCLOS/眨眼）。干净的是前四个。

## ⭐⭐ 本轮推荐：登台「歌词/和弦提词」

**一句话**：唱歌/弹琴/带敬拜时，下一句歌词、下一个和弦、当前在歌曲哪段，**浮在你眼前**；用 **Rokid Ring 轻点翻页**，双手不离琴、眼不离观众。

**为什么五个里最强**
- **手机做不了 + 耳机也做不了（唯一双否）**：手在乐器、眼在观众，掏手机=表演事故；而你正在**唱/奏**，音频提示会和你的声音撞——**必须是视觉**，这把耳机/录音笔彻底排除。**只有眼前 HUD 能做**，形态护城河最深。
- **痛点真实 + 大市场**：Adele/Taylor Swift/Streisand 都当众忘词翻车（Streisand 因此 27 年不敢登台）；职业音乐人 16.5–60% 有表演焦虑，忘词是首因。已有"提词地屏"（Robert Plant/Billy Strings 在用）、钢琴内置提词器（McCartney/Elton）证明需求。**最大盘是敬拜**：CCLI 23 万+ 教会、Planning Center 9 万+ 教会 / 每周 400 万志愿者，教会演示软件 ~1–2 亿美元市场。
- **硬件契合满分**：输出纯文本（歌词/和弦名），几个绿字，<350ms 蓝牙传，无图无色无 AR。正好**避开五线谱的坑**（密集图形，单色窄 HUD 显不了——所以"翻谱"证据虽最硬但硬件不匹配，放弃）。
- **白地确认**：Even/Meta/Rokid 的提词器**全瞄演讲者和视频博主，没一个做音乐/歌词/和弦**。
- **触发器现成**：Rokid 自带 BLE Ring，轻点翻页——比语音干净（唱歌时语音必误触），比脚踏更随身。

**注意**：① 触发**别用语音**（歌声=同源信号必误触），用 Ring/脚踏；② 别被"翻五线谱"诱惑（硬件显不了密集乐谱），主打**歌词 + 行内和弦**的文本提词。

## 次选
- **隐形计时器（当众讲话）**：最普适、**零"AI作弊"舆论风险**（它只是个钟）、硬件契合满分；但"哇"点弱（是个钟），且手表也能震一下，护城河没音乐深。
- **拍电阻→报数值 / 报尺寸悬浮（手+工具）**：真正用上摄像头（眼镜真超能力），手脏时手机物理失效，且把"单色不能显色"反转成优势（摄像头看色、HUD/语音报文本）；但受众偏窄。
- **3am 喂奶提示**：手机做不了最强（抱娃+黑灯+缺觉三杀）、情绪最戳；受众最窄，且贴近记录类 App。

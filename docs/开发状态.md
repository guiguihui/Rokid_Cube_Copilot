## [2026-06-06] 修复扫描解码：Craft takePhoto 返回 PNG 非 WebP
**背景/触发**：扫描页拍照报 `Failed to read WebP features: bitstream-error, chunk=unknown`。Craft 的 `takePhoto` 返回 **webcam 截图(PNG/JPEG)，非 WebP**，而我只用了 `decodeWebP`。另有无害警告 `set_data: key=mode has no bindings`。
**改了什么**：
- `lib/vendor/tiny-inflate.js` — **新增**，自洽 DEFLATE 解压(foliojs tiny-inflate, MIT, 无依赖)，UMD→ES。
- `lib/png.js` — **新增**，极简 PNG 解码(8-bit, colorType 0/2/4/6, 非隔行) → `{width,height,rgba}`，用 tiny-inflate。
- `pages/scan/index.ink` — `decodeFrame(photo)` 按 magic 字节分支：webp→decodeWebP、png→decodePng、jpeg→提示待支持、未知→报 mimeType+magic；debug 行加显示 mimeType。`mode` 改为实例变量(不进 data)消除 `has no bindings` 警告。
**为什么这么改**：不同运行时 takePhoto 编码不同，需按真实格式解码；PNG 是浏览器 canvas 截图默认。tiny-inflate 自洽适合 rquickjs（不像 upng 依赖 pako）。
**验证**：node 合成 PNG（逐行轮换 0~4 号 filter，RGBA/RGB 两类）`decodePng` 往返**精确 PASS**；全文件语法检查通过。Craft 真实拍照待用户验证。
**遗留/下一步**：若 Craft 实为 JPEG，按屏幕提示的 mimeType 再加 jpeg 解码器。
